システム仕様書: Obsidian Tasks Sync (v1.0)1. システム概要本システムは、ノートアプリ「Obsidian」のプラグイン「Obsidian Tasks」で管理されているタスクデータを、スマートフォンなどのモバイルブラウザからシームレスに閲覧・編集・追加するための、サーバーレスのシングルページアプリケーション（SPA）です。2. アーキテクチャと技術スタック1つのHTMLファイルのみで完結し、バックエンドサーバーを必要としません。フロントエンド: HTML5, React 18 (CDN経由でBabelによるブラウザ内トランスパイル実行)スタイリング: Tailwind CSS (CDN経由)アイコン: Lucide React (CDN経由)データ同期・ストレージ: GitHub REST API (Contents API)ローカル保存: Web Storage API (localStorage - GitHub設定情報の保持)PWA実装: JavaScript Blob を用いた Manifest および Service Worker の動的生成3. 主要機能タスク表示とグループ化未完了タスクを期日ベースで自動分類（Overdue[期限切れ], Today[今日], Upcoming[明日以降], No Due Date[期日なし]）。完了済みタスク（Completed）の折りたたみ表示。タスク操作新規タスクの追加（タグ自動抽出機能付き）。タスクの編集（タイトル、期日、繰り返し、優先度、タグ、メモ）。タスクの完了/未完了の切り替え、削除。スマートフィルター（タグ絞り込み）現在使用されているすべてのタグを自動抽出し、ワンタップで絞り込みが可能。自動同期（オートセーブ）アプリ起動時の自動フェッチ（Pull）。タスク操作（追加・編集・状態変更）から2秒後に自動的にGitHubへ変更をプッシュ（Debounce処理によりAPIコール数を削減）。PWA（Progressive Web App）対応スマートフォンブラウザの「ホーム画面に追加」からインストール可能。ネイティブアプリのようなフルスクリーンUIを提供。4. データフォーマット仕様 (Obsidian Tasks 互換)アプリ内部のJSONデータは、同期時に以下のルールに従ってMarkdownのプレーンテキストにパース/シリアライズされます。【基本構文】- [ステータス] タスク名 ⏫(優先度) 🔁(繰り返し) 📅(期日) #(タグ)    メモ内容(4スペースインデント)ステータス: 未完了は [ ]、完了は [x]。優先度 (Priority): Highは ⏫、Mediumは 🔼、Lowは 🔽。期日 (Due Date): 📅 YYYY-MM-DD 形式。繰り返し (Recurrence): 🔁 every day などの文字列。タグ (Tags): #tagname の形式。複数付与可能。メモ (Notes): タスクの直下にインデント（スペース4つ）されて配置される文字列。URLはアプリ上で自動的にクリッカブルリンク化される。5. 連携要件 (GitHub設定)アプリを利用するには、画面左メニューの「GitHub Sync Settings」から以下の情報を設定する必要があります。Personal Access Token (PAT): GitHubにて発行した repo スコープを持つトークン。Owner: 対象リポジトリの所有者名（ユーザー名）。Repository Name: 対象のリポジトリ名。File Path: 同期対象となるMarkdownファイルのパス（例: Tasks.md または 01_Vault/Tasks.md）。6. 制限事項と今後の拡張性通信制約: GitHub APIのレート制限（通常1時間あたり5000回）に依存します。Debounce処理により通常利用では到達しませんが、留意が必要です。コンフリクト処理: 複数デバイスからの同時編集によるコンフリクト（競合）を解決するマージ機能は未実装です。最後にPushしたデータが優先されます（上書き）。
